Method Detail

refreshCell

void refreshCell(<number> chunkY, <number> chunkX, <number> z, <number> y, <number> x)
    Tells the renderer that something changed about the given cell. It can be for different reasons: a new block, a block changed type, a block was destroyed, a player on the given cell moved.
    Parameters:
      chunkY - chunk Y coordinate.
      chunkX - chunk X coordinate.
      z - cell Z coordinate.
      y - cell Y coordinate.
      x - cell X coordinate.

refreshChunk

void refreshChunk(<number> chunkY, <number> chunkX)
    Similar to refreshCell, but for a chunk. This tells the renderer that something significant changed in this chunk (e.g. it was reloaded) and the renderer should repaint this chunk. This is only called when multiple cells have changed in the given chunk.
    Parameters:
      chunkY - chunk Y coordinate.
      chunkX - chunk X coordinate.

reloadVisibleParts

void reloadVisibleParts()
    Tells the renderer to determine what chunks are visible to the player (or close). The renderer should then call reloadChunk on each of the visible chunks, followed by a call to listenChunks. This function is basically called when the client is online either for the first time, or after a deconnection. TODO: this function is missnamed.

setDirty

void setDirty()
    Tells the renderer to repaint the whole screen. This happens when the client goes offline. TODO: this function is at least missnamed. The library should not decide when the renderer have to repaint.
